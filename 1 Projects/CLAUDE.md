## Five PMF/GTM project types

Every Running project is tagged ONE of these in frontmatter (`type: land | expand | speak | write | partner`). If it can't be tagged, it's not a project.

| Type | Goal | Examples |
|---|---|---|
| **LAND** | OSS install (top of funnel) | README polish, HN/Reddit, dbt Slack share, demo video |
| **EXPAND** | Install → AutoFyn intro | Cal link, post-install survey, follow-up DMs |
| **SPEAK** | Credibility events | dbt Summit CFP, MCPCon, podcast slots |
| **WRITE** | Content that sells | Blog deep-dives, X threads, LinkedIn |
| **PARTNER** | Multiply via co-hosts | Work-Bench, Hex, Hightouch, Helicone, dbt Labs |

Internal closure work (e.g., [Multiplyr Wind Down](1%20Next%20Projects/Multiplyr%20Wind%20Down/), [Fundraising](1%20Next%20Projects/SignalPilot%20Fundraising.md)) uses the [Generic Project Template](../Templates/Generic%20Project%20Template.md) instead.

## Project structure

Use [PMF Project Template](../Templates/PMF%20Project%20Template.md) (GTM) or [Generic Project Template](../Templates/Generic%20Project%20Template.md) (internal).

Required sections:
- **Goals** — measurable, aligned with the [SignalPilot thesis](0%20Running%20Projects/SignalPilot%20New%20Direction/index.md).
- **Deliverables** — shareable assets (the things you ship), not tasks.
- **Tasks** — atomic, <2 min where possible.
- **Hypothesis + Kill criteria** (PMF projects only) — what makes this win or get cut.
- **Completed** — done tasks land here. Builds your shipped-evidence trail.

## Lifecycle

```
2 Deferred  →  1 Next  →  0 Running  →  4 Archive
```

- **Deferred → Next:** "Will I start within 1–2 weeks?"
- **Next → Running:** "Am I starting THIS week?" (decided every Sunday)
- **Running → Archive:** complete OR no deliverable in 14 days OR no longer aligned

---

## Hard limits

- **Max 3 Running. Ideal 1–2.** More than 3 = context-switching to death.
- Any project without a shipped deliverable in 7 days gets demoted to Next.
- 14 days no shipped deliverable → archive.
