# Templates — Default Practices, Codified

> **Function:** Eliminate decision fatigue. Use mental energy on content, not structure.

---

## Current templates

- **[PMF Project Template](PMF%20Project%20Template.md)** — for Land/Expand/Speak/Write/Partner projects. Includes type, hypothesis, kill criteria, success metrics.
- **[Generic Project Template](Generic%20Project%20Template.md)** — for internal/closure work (e.g., Multiplyr Wind Down, Fundraising).
- **[Weekly Review Template](Weekly%20Review%20Template.md)** — Sunday 30-min ritual.
- **[Day Planner Template](Day%20Planner%20Template.md)** — daily execution structure.
- **[Generic Idea Template](Generic%20Idea%20Template.md)** — for braindump capture.

---

## Which to use

| Need | Template |
|---|---|
| New PMF/GTM project (drives funnel) | [PMF Project Template](PMF%20Project%20Template.md) |
| Internal project (no funnel impact) | [Generic Project Template](Generic%20Project%20Template.md) |
| Weekly review on Sunday | [Weekly Review Template](Weekly%20Review%20Template.md) |
| New day | [Day Planner Template](Day%20Planner%20Template.md) |
| Random idea / capture | [Generic Idea Template](Generic%20Idea%20Template.md) |

If you're creating a structure 3+ times, it should be a template. If a template has unused sections after 3 uses, prune them.

---

## AI behavior

- When user asks for a new project, use the right template (ask which if ambiguous).
- Day Planner: respect start-time-only convention.
- Weekly Review: drive the user through the time-boxed sections; don't let them skip the funnel metrics block.
- Quarterly maintenance: if a template hasn't been used in a quarter, propose pruning or merging.

---

*Templates make the right thing easy and the wrong thing hard.*
