# Weekly Work Logs — Funnel Metrics + Honest Review

> **Function:** Daily progress trail + Sunday metrics block. The accountability surface for the credibility window.

---

## File convention

- One file per week: `YYYY-WXX-<start>_<end>.md` (e.g., `2026-W18-Apr27_May03.md`).
- Created Monday morning. Copy structure from previous week.
- Archive logs older than 4 weeks to [4 Archive](../4%20Archive/).

---

## Daily entry (3–5 bullets, end of work session)

```
### Mon Apr 27
- **Projects:** [SignalPilot Fundraising], [Multiplyr Wind Down]
- **Shipped:** 5 investor DMs, blog draft v1, plugin install fix
- **Meetings:** Work-Bench Jon Lehr 1:1, Daniel sync
- **Funnel:** +12 GitHub stars, +3 plugin installs, 1 AutoFyn intro booked
- **Archived:** [[Stale Project X]]
```

Don't duplicate project content. Link.

## Coach's review (Sunday evening, brutally honest)

- **Rating:** A / B / C / D
- **Worked:** 1–2 things
- **Hard truths:** misalignment, time spent on internal polish vs outbound
- **Next-week correction:** one specific change

> **External vs internal balance check.** Target: 60% external (DMs, demos, content, dinners) / 40% internal (code, specs, docs). If outbound metrics are empty → flag the week as misaligned.

---

## AI behavior — CRITICAL

- **End of session: update today's entry.** Don't wait.
- When helping with Day Planners: log to the corresponding day in the current week's log.
- On Sunday: prompt for funnel metrics + coach's review if missing.
- If outbound metrics are flat 2 weeks: **call it.** Lines of code shipped ≠ company progress.
- Pre-pivot framing in entries (notebook SignalPilot, signalpilot-cli) → flag, don't propagate.
